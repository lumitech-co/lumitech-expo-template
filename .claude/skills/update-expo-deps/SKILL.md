# Update Expo Dependencies

Safely update dependencies for `apps/expo` in the Craftwork monorepo. Commits after each phase so changes are easy to revert.

## Scope

- **fix-only** (default): Run `expo install --fix` to align Expo-managed packages to current SDK
- **third-party**: Fix + update non-Expo packages to latest minor/patch
- **full-sdk**: Upgrade the Expo SDK version (bigger change, avoid before releases)

## Step 1: Prune Stale Excludes

Before updating anything, check `expo.install.exclude` in `apps/expo/package.json`. Excludes are **temporary** — they exist only because we updated a package beyond what the current SDK expects. Over time, SDK updates catch up and the exclude becomes unnecessary.

For each excluded package (other than the permanent ones: `react`, `react-dom`):

1. Run `cd apps/expo && pnpm with-env npx expo install --check` to see what the SDK expects
2. If the SDK now expects a version equal to or higher than what we have, **remove it from the exclude list**
3. If the SDK still expects an older version, keep the exclude

**Goal: keep the exclude list as small as possible.** Only `react` and `react-dom` should be permanent excludes (they're catalog-managed). Everything else should be removed as soon as the SDK catches up.

## Step 2: Expo SDK Alignment

Run `expo install --fix` to ensure all Expo-ecosystem packages match the current SDK version:

```bash
cd apps/expo && pnpm with-env npx expo install --fix
```

This updates packages like `react-native-screens`, `react-native-reanimated`, `react-native-gesture-handler`, etc. to their SDK-compatible versions.

**Commit after this step** before proceeding.

If scope is `fix-only`, skip to Step 6.

## Step 3: Identify Outdated Packages

```bash
pnpm --filter @craftwork/expo outdated
```

Categorize every package into one of these buckets:

| Category | Action | Examples |
|---|---|---|
| **Safe minor/patch** | Update | `posthog-react-native`, `react-native-svg`, `lucide-react-native` |
| **Expo-managed (SDK-pinned)** | Skip (already handled in Step 2) | `react-native-reanimated`, `react-native-screens`, `expo-*` |
| **Catalog-managed** | Skip (monorepo-wide, defined in `pnpm-workspace.yaml`) | `react`, `react-dom`, `zod`, `eslint`, `prettier` |
| **Major version bump** | Skip unless explicitly requested | `react-native-mmkv` 3->4, `tailwindcss` 3->4 |

**Check `pnpm-workspace.yaml`** for `catalog:` entries before updating. Packages using `catalog:` in their version field are managed at the monorepo level and must not be updated per-app.

## Step 4: Update Third-Party Packages

Update safe packages using pnpm from the repo root:

```bash
pnpm --filter @craftwork/expo add package1@version package2@version ...
```

For packages with tilde ranges in package.json (like `@sentry/react-native: ~7.8.0`), update the range to target the new version (e.g., `~7.12.0`).

Present a summary table of all updates to the user before proceeding.

## Step 5: Add Excludes for Newly Updated Packages

Any package updated beyond what Expo SDK expects must be added to `expo.install.exclude` in `apps/expo/package.json` so `expo doctor` and `expo install --check` don't flag them.

These excludes are **temporary** and will be pruned in Step 1 of the next update cycle once the SDK catches up.

## Step 6: Verify

Run typecheck and tests first:

```bash
pnpm --filter @craftwork/expo typecheck
pnpm --filter @craftwork/expo test
```

Fix any failures before proceeding.

## Step 7: Commit

Commit the dependency updates with a descriptive message listing all updated packages and their version changes.

## Step 8: Final Health Check

As the very last step, run expo doctor and install check to confirm everything is clean:

```bash
cd apps/expo && pnpm with-env npx expo-doctor
cd apps/expo && pnpm with-env npx expo install --check
```

**Both must pass.** The only acceptable `expo doctor` failure is the non-CNG project warning about native folders — if native folders (`android/`, `ios/`) exist locally and trigger this, they can be safely deleted (they're gitignored and regenerated by prebuild).

If either check fails, fix the issue and amend the commit before finishing.

## Common Pitfalls

- **Don't update `tailwindcss` to v4** - NativeWind is tied to v3
- **Don't update `react-native` directly** - it's tied to the Expo SDK version
- **Don't update `jest`/`babel-jest` across major versions** without verifying test infrastructure compatibility
- **Check for `minimumReleaseAge`** in `pnpm-workspace.yaml` - pnpm may block very recent releases (currently 48 hours)
- **Don't let excludes accumulate** - prune them every update cycle (Step 1)
